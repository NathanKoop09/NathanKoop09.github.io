Slopehou

Technical Details

Due to the shear amount of kill bricks that can be thrown at you during the higher speeds several optimizations had to be made to prevent any lag, as even a small amount of lag will cause player death.
The first is spawn pooling which prevents the expensive process of creating new objects and prevents the game from rendering irrelevant obstacles that are now behind the player.
The kill bricks are ordered in memory in such a way that computation for collision and grazing of a kill brick is only performed on those that are close to the player, all other bricks are ignored.
As the game gets faster, to ensure the survivability of all obstacles, the horizontal speed scales with the forward speed. This ensures that so long as the player can react quickly enough they can maneuver through the obstacles using the same path as before.

Project Description

An infinite runner style game with the difficulty cranked up in order to make the game feel more like a 3d bullet hell game.
There are nine different patterns to weave your way through, each with three different variations. If you master each one you should have no problem facing the ever faster nature of the procedurally generated course.
