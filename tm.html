Termino

Technical Details

A bullet hell game written in Python where the game itself is printed to the terminal window. Each frame the window is cleared and the next frame is printed. The game uses an object oriented approach to deal with the five different spell cards and keeping track of the bullets on screen and their momentum.
Termino uses multi-threading in order to allow the input manager, bullet manager, and music player to all operate in parallel which results in an overall smoother gameplay experience.
The game has two different soundtracks it can play directly from the terminal, one for the menu and another for the game which are played using the winsound library.

Project Description

Play through an endless bullet hell game contained entirely in the terminal. There are five spell cards to survive, the more cards you survive, the faster the game goes. How far can you make it in the land of Termino?
